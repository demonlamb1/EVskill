## 设计逻辑

- **主工具默认是"工具箱模式"，不是强制的完整诊断流程**——这和"处境解"不一样。原因：交易场景里，用户大概率是带着一个具体问题来的（"这笔交易该不该做""止损设多少"），而不是每次都想从头走一遍完整评估。只有用户明确说"帮我完整评估"时，才按六个节点顺序走一遍。
- **常见误区、复盘迭代独立成skill**，因为这两个场景用户可能完全脱离"评估一笔具体交易"这个语境单独触发（复盘是交易完成之后的动作，常见误区可能在对话任何时候被点破）。
- **产品与工具节点保持纯信息，没有加任何自我推广**，按你的要求处理。
- 三个skill各自独立，靠description里的触发词自然分工，不需要路由器串联——如果在支持Skills目录的Agent（Claude Code/Codex等）环境下同时安装了三个，可以互相提及；单独安装其中一个也能正常工作。

## 安装

三个文件夹分别放入对应Agent的skills目录即可。这一版没有做面向豆包等消费级对话工具的合并版——如果需要，参考"处境解"包里的合并方式，去掉SKILL.md的YAML frontmatter和文件路径引用，把references内容按顺序拼接即可。
